## OpenClaw tool-call schema hotpatch for Kimi / GPT-5.4 (2026-03-26 UTC)

### Problem
- Claude Opus handled alias-heavy tool schemas fine, but Kimi and GPT-5.4 were more literal.
- OpenClaw flattens `anyOf`/`oneOf` tool parameter schemas in `/usr/lib/node_modules/openclaw/dist/pi-embedded-BaSvmUpW.js`.
- The flattening logic only preserved `required` keys that were required in **all** variants.
- For alias-style tools like `read`, `write`, `edit`, `exec`, and memory-style tools, that often produced `required: []` after flattening.
- Result: some models emitted empty tool calls, looped, or failed to provide the actual path/query/command parameter.

### Runtime fix applied
- Changed Kimi fallback in `/root/.openclaw/openclaw.json` from `kimi/k2p5` to `kimi/kimi-code`.
- Hotpatched `/usr/lib/node_modules/openclaw/dist/pi-embedded-BaSvmUpW.js` so when flattened schemas would otherwise have no required params, OpenClaw now adds a fallback required field using a preferred alias order:
  - `path`, `file`, `command`, `query`, `prompt`, `sessionKey`, `url`, `image`
  - else first merged property key
- Restarted the OpenClaw gateway successfully after patching.

### Status
- Gateway healthy after restart.
- Behavior change is active now.
- This is a **runtime patch**, not a durable upstream source commit yet.

### Durable follow-up needed
1. Port the fix into actual OpenClaw source and upstream it / preserve it through rebuilds.
2. Prefer a cleaner per-tool alias-preference strategy if source ownership allows.
3. Keep this note as the breadcrumb if a package update overwrites the patch.
