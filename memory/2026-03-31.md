### SOP: When Sonnet (or any primary model) stops working

**Signs:**
- Messages still delivering but replies feel different (faster/shorter/less capable)
- `/status` shows unexpected model (e.g. gpt-5.4 when Sonnet should be primary)
- You see 503 / auth errors in logs but responses still come (fallback silently activated)
- Claude Desktop works fine but Clawd doesn't

**Step 1 — Verify what model is actually being used:**
```bash
python3 - <<'PY'
import json
p='/root/.openclaw/agents/main/sessions/sessions.json'
with open(p) as f: data=json.load(f)
k='agent:main:telegram:direct:8001393940'
e=data.get(k,{})
print(json.dumps({x:e.get(x) for x in ['model','provider','authProfileOverride','authProfileOverrideSource','authProfileOverrideCompactionCount']}, indent=2))
PY
```

**Step 2 — Clear the stale override if present:**
```bash
python3 - <<'PY'
import json, shutil, datetime
p='/root/.openclaw/agents/main/sessions/sessions.json'
with open(p) as f: data=json.load(f)
k='agent:main:telegram:direct:8001393940'
e=data[k]
shutil.copy2(p, p + '.bak-' + datetime.datetime.now(datetime.UTC).strftime('%Y-%m-%dT%H%M%SZ'))
for field in ['authProfileOverride','authProfileOverrideSource','authProfileOverrideCompactionCount','model','provider']:
    e.pop(field, None)
with open(p,'w') as f: json.dump(data,f,indent=2); f.write('\n')
print('Cleared. Send a test message — no restart needed.')
PY
```

**Step 3 — If still broken after clearing session, check auth profile:**
```bash
cat /root/.openclaw/agents/main/agent/auth-profiles.json
```
Look for `anthropic:tmw` or `anthropic:default`. If missing or token looks truncated/stale, re-run:
```bash
openclaw models auth setup-token
```

**Step 4 — If 503 / rate limit is the actual issue (not session override):**
Force a temporary model switch with:
```
/model openai-codex/gpt-5.4
```
Then switch back when Anthropic recovers:
```
/model anthropic/claude-sonnet-4-6
```
