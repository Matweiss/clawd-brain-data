# General Context Routing

Use relationship-first recall beyond events.

## For broader non-trivial questions
When Mat asks for advice, what to do next, how to think about something, or how to prioritize:

1. Search memory for the relevant domain:
   - active projects
   - recent blockers
   - current priorities
   - known preferences
   - prior decisions
   - people involved
2. Infer the most likely active thread.
3. Answer for that thread rather than giving generic advice.

## Examples
- "What should I focus on today?" -> combine schedule + active projects + recent blockers
- "How should I think about this offer?" -> recall prior deal/job/company context
- "What should I do next with Sarah's agent?" -> recall current Arty/ManyChat/dashboard state
- "What am I forgetting?" -> search recent commitments, blockers, and logistics

## Rule
If continuity would materially improve the answer, recall first.
